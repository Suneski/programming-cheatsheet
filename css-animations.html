CSS ANIMATIONS

With the use of CSS, you can animate objects on your page (like Clyde above). CSS is deceptively versatile when it comes to setting an animation.

Let's set up a simple animation with our old friend, Mr. Red Square.

  1. First, create your object (in this case, the div that is the red square).
  2. Within the object's selector, set your "animation-name:" to something of your choosing. For this example, I've named mine 'to-mr-blue-box'
  3. Next, select your animation's duration with "animation-duration:" (include the unit 's' for seconds or 'ms' for milliseconds). For this example, I have my animation duration set to 5s
  4. From here, you can include several different animation options (full lists included in resources below). These include, but are not limited to, setting how many times the animation will occur, direction, immediacy of animation, and more! Since we are building a simple color-changing square, we'll hold off on most of these.

    NOTE: I will be including the "animation-iteration-count" property, and setting it to infinite.

  5. Now that we have added our basic square, we need to create a new rule, a keyframe rule. To do this, use @keyframes, along with the name of your animation, followed by squiggle brackets.
  6. Between your squiggle brackets, you will want to set a from{} and a to{}
  7. Depending on your animation, you may or may not have something between your from-squigglies. This could include location on a page or other initial properties. For this example, we'll just be leaving it blank.
  8. The to{} is where it gets exciting. This is the final state of your animation. This is what you want your animation "to" become. For this example, I've set the background color to blue, but I could also change numerous other properties, including size and location.

    NOTE: Instead of to/from, you can also set percentages, such as 0%{}, 25%{}, 100%{} etc. This allows for differing transitions at different points of the animation.

  9. Voila! A Mr. Red Box is now Mr. Blue Box! And here he is, along with his code!

Hold up. Dial it back a bit. So when do you use CSS animations, and when do you use CSS transition?

It depends on what you need!

If your animation needs to start as soon as the page loads (without some sort of provocation), or if it is more than going from one state to another, then use CSS animation.

If it's a simple transition from one state to another, or is something that requires some user interaction (such as a hover or a click), then CSS transition is good option
